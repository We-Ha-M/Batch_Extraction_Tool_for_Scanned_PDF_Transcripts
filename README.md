# 扫描版 PDF 成绩单批量提取工具

这是一个面向 **高校学委、教务助理** 的轻量级 Python 工具，能够自动从扫描版 PDF 成绩单中识别学生信息（学号、姓名）以及指定课程的成绩，并将所有结果汇总到一份 CSV 表格中，告别手动抄录的繁琐工作。

## 项目简介

每到学期末，学委总需要收集全班同学的纸质成绩单（扫描件），并逐份录入 Excel。本工具利用 **OCR 技术**（基于 PaddleOCR）自动解析 PDF 中的文字与数字，智能匹配课程名称与成绩，最终生成一个清晰的汇总表格。

- **输入**：一个文件夹内所有扫描版 PDF 成绩单（每份成绩单应包含学生的学号、姓名以及各门课程的成绩）
- **输出**：一个 CSV 文件（可用 Excel 打开），包含“学号”“姓名”以及用户指定的课程列表，每个学生的成绩对应填写
- **核心算法**：基于 PaddleOCR 的文字识别 + 基于坐标与文本匹配的课程定位 + 自适应成绩列识别

## 功能特点

- ✅ 支持 **扫描版 PDF**（图片内嵌文字）  
- ✅ 自动提取 **学号**（10 位数字）和 **姓名**（中文字符）  
- ✅ 支持自定义 **课程列表**，按顺序输出成绩  
- ✅ 智能匹配课程名称（支持简繁、括号、空格差异）  
- ✅ 自动定位“成绩”列，准确提取分数（包括补考标注和五级制文本成绩）  
- ✅ 批量处理整个文件夹的 PDF  
- ✅ 输出 **CSV 格式**，可直接用 Excel 打开  
- ✅ 除初次运行外完全离线运行，无需联网（模型初次运行下载后本地使用）

## 快速开始

### 0. 换源

在命令行中执行以下命令更换清华源：

```bash
python -m pip install --upgrade pip
pip config set global.index-url https://mirrors.tuna.tsinghua.edu.cn/pypi/web/simple
```

### 1. 安装依赖

打开

在命令行中执行以下命令安装所需 Python 包：

```bash
pip install paddleocr==2.7.3 paddlepaddle==2.6.2 "numpy<2" pdf2image Pillow
```

### 2. 下载 OCR 模型（自动完成）

首次运行程序时，工具会自动下载所需的 OCR 模型（检测、识别、方向分类），并解压到 `MODEL_DIR` 指定的目录。

你可以在脚本的 **用户配置区** 中修改 `MODEL_DIR` 的路径。

### 3. 配置脚本参数

打开脚本文件，根据你的实际情况修改以下配置：

```python
# 需要提取的课程列表（按输出顺序排列）
COURSE_LIST = [
    "数据结构课程设计",
    "数据结构",
    "离散数学",
    ...
]

# PDF 文件夹路径（存放所有成绩单扫描件）
PDF_FOLDER = r"./pdfs"

# 输出 CSV 文件名
OUTPUT_FILE = "成绩汇总.csv"

# Poppler 的 bin 目录路径（Windows 必须填写）
POPPLER_PATH = r"./Poppler/Library/bin"

# 模型存放目录（纯英文路径）
MODEL_DIR = r"./Poppler/paddleocr_models"

# 图片识别 DPI（一般无需修改）
DPI = 300

# 行、列容差（用于文本分组，一般无需修改）
ROW_Y_TOL = 20
COL_X_TOL = 80
```

- **课程列表**：建议按照成绩单上实际的课程名称填写，支持模糊匹配（例如“大学物理(2)”与“大学物理（2）”均可匹配）。
- **PDF 文件夹**：只处理该文件夹下的 `.pdf` 文件，不会递归子目录。

### 4. 运行脚本

在命令行中执行：

```bash
python main.py
```

程序会依次处理每个 PDF，输出每份成绩单的识别结果（学号、姓名、各科成绩），最后生成 CSV 文件。

### 5. 查看结果

用 Excel 打开生成的 CSV 文件，即可得到汇总成绩表。如果成绩缺失，对应单元格留空。

## 配置详解

### 课程匹配规则

脚本会尝试将 OCR 识别的课程名与你提供的 `COURSE_LIST` 进行匹配。匹配规则如下：
- 去除空格、统一括号（全角/半角）、去除“+”“十”等符号
- 若完全一致则直接匹配

### 成绩提取策略

1. 先识别表格中“成绩”列的 **X 坐标**，然后根据该坐标获取对应分数。
2. 如果上述方法失败，则查找“学年学期”等日期字段，取日期前面的文本作为成绩。
3. 若 OCR 识别出“补考”，会自动追加到成绩后（例如“60补考”）。

### 常见问题

#### Q1: 运行时报错 “缺少 pdf2image” 或 “缺少 paddleocr”
- 请确保已正确安装所有依赖（见“安装依赖”一节）。

#### Q2: 提示 “PDF 转图片失败”
- 可能原因是 Poppler 路径设置错误。请检查 `POPPLER_PATH` 是否指向正确的 `bin` 目录，并且该目录下存在 `pdftoppm.exe`（Windows）等文件。

#### Q3: 模型下载失败或解压错误
- 网络问题可能导致下载中断，可以手动下载模型文件并放置到 `MODEL_DIR`。模型地址如下：
  - 检测模型：https://paddleocr.bj.bcebos.com/PP-OCRv4/chinese/ch_PP-OCRv4_det_infer.tar
  - 识别模型：https://paddleocr.bj.bcebos.com/PP-OCRv4/chinese/ch_PP-OCRv4_rec_infer.tar
  - 分类模型：https://paddleocr.bj.bcebos.com/dygraph_v2.0/ch/ch_ppocr_mobile_v2.0_cls_infer.tar
- 请确保 `MODEL_DIR` 是纯英文路径，且程序有写入权限。

#### Q4: 学号或姓名识别不准
- 学号目前匹配 10 位数字，如果学号长度不同，可修改脚本中的正则表达式 `\d{10}`。
- 姓名识别依赖“姓名”关键词，如果成绩单上写的是“学生姓名”或排版特殊，可以调整 `_student_info` 方法。

#### Q5: 成绩提取不全或匹配错误
- 可以尝试调整 `ROW_Y_TOL` 和 `COL_X_TOL` 的值，分别控制行合并的 Y 坐标容差和列定位的 X 坐标容差。
- 若成绩单的“成绩”列不在常规位置，可考虑修改 `_grade_in_row` 中的列定位逻辑。

## 注意事项

- 本工具基于 OCR 技术，识别准确率受扫描质量、字体、排版影响，建议先使用少量成绩单测试，确认效果后再批量处理。
- 运行环境建议为 Python 3.8+。
- 模型下载和解压需要一定的磁盘空间（约 50 MB）。
- 处理大量 PDF 时，程序可能会占用较多内存，请确保计算机资源充足。
